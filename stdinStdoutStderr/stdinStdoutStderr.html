stdin (Standard Input):

É o canal por onde o programa recebe entrada de dados, geralmente do teclado ou de outro programa.
Pense como: "O que o usuário digita ou o que outro processo envia para o meu programa."
Exemplo: Quando você digita algo no terminal e o programa lê, isso vem do stdin.

stdout (Standard Output):

É o canal por onde o programa envia sua saída normal, como resultados ou mensagens para o usuário.
Geralmente, isso aparece no terminal/console.
Pense como: "O que o programa quer mostrar para o usuário como resultado."

stderr (Standard Error):

É o canal usado para enviar mensagens de erro ou diagnósticos.
Também aparece no terminal, mas é separado do stdout para facilitar a distinção entre resultados normais e erros.
Pense como: "O que o programa diz quando algo dá errado."

Por que isso é importante?
                Interação com o usuário: stdin permite que seu programa receba dados do usuário (como formulários no terminal). 
                stdout mostra resultados claros.

                Depuração: stderr ajuda a identificar problemas sem misturar mensagens de erro com a saída normal do programa.

                Integração com outros programas: Em sistemas como Linux, você pode redirecionar stdin, 

                stdout e stderr para arquivos ou outros programas (usando >, |, etc.), o que é poderoso para automação.

                Boas práticas: Separar erros (stderr) de saídas normais (stdout) melhora a legibilidade e a manutenção do código.

stderr:Stack de Erros (Stack Trace) 

            

              É um relatório gerado quando ocorre um erro em um programa, mostrando a sequência de chamadas de funções que levou ao erro.

              Pense nisso como um "mapa" que rastreia o caminho que o programa percorreu até o ponto onde algo deu errado.

              Ele lista as funções chamadas, em ordem, desde o início da execução até o momento do erro, incluindo o arquivo, a linha e, às vezes, a coluna onde o problema ocorreu.
            

        
Por que é importante?

            
                Depuração: Ajuda a identificar onde e por que o erro aconteceu, mostrando o contexto (quais funções foram chamadas).
                Rastreabilidade: Permite seguir o "rastro" do código para entender a sequência de eventos que causou o problema.
                Correção: Com o stack trace, você pode ir diretamente ao arquivo e linha problemáticos para corrigir o código.
                Integração com ferramentas: Ferramentas de depuração (como IDEs ou logs) usam o stack trace para destacar o erro no código.
            

        
Ele inclui

            
                O tipo de erro (ex.: "TypeError", "NullPointerException").
                Uma mensagem descritiva (ex.: "Cannot read property 'x' of undefined").
                Uma lista de chamadas de funções, com detalhes como arquivo, linha e coluna.
            

        
Tipos de Erros:
   
            

                O tipo de erro é a categoria ou classificação do erro que ocorreu no programa. 

                Cada linguagem de programação define seus próprios tipos de erros, mas eles geralmente se enquadram em grupos comuns.

                O tipo de erro é indicado no início da mensagem de erro (no stderr) e ajuda a entender a natureza do problema.

            

        
Por que é importante?
  
            
                Diagnóstico rápido: O tipo de erro indica imediatamente o que deu errado (ex.: acesso a uma variável nula, sintaxe incorreta, etc.).
                Tratamento de erros: Em muitas linguagens, você pode capturar erros específicos (usando try/catch ou equivalentes) com base no tipo de erro.
                Prevenção: Conhecer os tipos de erros comuns ajuda a escrever código mais robusto, com validações para evitar esses problemas.
            

        
Tipos de erros comuns (genéricos e aplicáveis a várias linguagens)
  
            

                Erro de Sintaxe > Exemplo: SyntaxError: unexpected token, SyntaxError: missing ; before statement, ou SyntaxError: expected ':'.

                Erro de Tempo de Execução > Exemplo: ArithmeticException: division by zero, RuntimeError: invalid operation, ou Segmentation fault.

                Erro Lógico > Nenhuma mensagem (resultados incorretos);

                Erro de Tipo > Exemplo: TypeError: unsupported operand type(s), TypeError: cannot concatenate 'str' and 'int' objects, ou TypeError: 'undefined' is not a function.

                Erro de Referência > Exemplo: ReferenceError: variable is not defined, NameError: name 'x' is not defined, ou NullPointerException.

                Erro de Recursos (Resource Error) > Exemplo: FileNotFoundError: No such file or directory, IOException: cannot open file, ou NetworkError: connection refused.
